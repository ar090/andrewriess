Projects

theGunGuy.com

Fully functional, and currently operational e-commerce store built primarily in Python using Django as a framework. The site follows a traditional drop shipping business plan which requires real-time inventory updates scheduled with Celery and Raw MySQL. Multistage software development lifecycle featuring local dev, staging network with sample data, and versioned production. Deployed with growth in mind utilizing easily scalable AWS databases, services and storage. Some notable integrations on theGunGuy.com include Algolia for their instant search service, and Authorize.net to customize a unique payment system while remaining PCI compliant. From an administrative perspective, the site is completely manageable from a non-technical perspective. Every user action and flow is accessible in an admin dashboard. Things like viewing user and product statistics, and links which execute python scripts to perform useful actions like charge a credit card were integrated as well.

Gunavail.com

This is a nifty data compilation, and normalization API that I built and ultimately stuck a front end on. Gunavail takes data from several distributors in any way possible (XML, CSV, TXT, scraping, yes even SOAP) to ultimately be normalized and combined into a master MySQL database for real-time inventory representation. This database is connected to a Flask API which feeds search results, data and other useful information about updates to customers. Celery is used to schedule all the tasks which update catalog information on 15 minute and weekly intervals.

Hubsy (the Hubspot Bot)

Hubsy was built as a submission for Amazon's Chatbot Challenge 2017 seeking to redefine the away users traditionally manage their HubSpot account. For Hubsy’s conversational interface and to execute complex commands within the HubSpot API, AWS – Lex was used in sync with AWS – Lambda. Users flow through conversation while variables are stored within Lex and transmitted to the HubSpot API via Lambda functions written in Node.js. Hubsy’s infrastructure was all managed via terraform and he is deployed on Facebook, Slack and AWS Alexa.
