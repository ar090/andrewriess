Projects

theGunGuy.com

Fully functional, and currently operational e-commerce store built primarily in Python’s Django framework. The site follows a traditional drop shipping business model which requires real-time inventory updates scripted together with Python, MySQL and scheduled with Celery. I use a complete multistage software development lifecycle featuring local dev, a staging server with sample data, and versioned production. It was deployed with growth in mind and features easily scalable AWS databases, services and storage. Some notable integrations on theGunGuy.com include Algolia for an instant search service, and Authorize.net to customize a unique payment process while remaining PCI compliant. From an administrative perspective, the site is completely manageable from a non-technical perspective. Every user action and flow is accessible in an admin dashboard. Several Python functions address all kinds of routine management duties. Things like viewing user and product statistics, charging credit cards, updating product or dealer info, and syncing data with 3rd party integrations are all easily accessible through the admin portal.

Gunavail.com

This started as a nifty data compilation, and normalization API that I built primarily for a local business to improve an internal process. I ultimately ended up sticking a quick JavaScript/JQuery front end on it so that the database could be easily searched outside of just the API. Gunavail takes data from several distributors in any way possible (XML, CSV, TXT, scraping, and yes even SOAP) to ultimately be normalized and combined into a master MySQL database for real-time inventory representation. This database is connected to a Flask API which feeds search results, data and other useful information about updates to customers. Celery is used to schedule all of the micro services that update catalog information continuously on 15 minute and weekly intervals. Gunavail was written with a ton of raw MySQL and Python to efficiently tear through hundreds of thousands of records on a routine basis. The tool is hosted in AWS and delivers content regularly to other dependent projects.

Hubsy (the Hubspot Bot)

Hubsy was built as a submission for Amazon's Chatbot Challenge 2017 seeking to redefine the way users traditionally manage their HubSpot account. For Hubsy’s conversational interface and to execute complex commands within the HubSpot API, AWS – Lex was used in sync with AWS – Lambda. Users flow through conversation while variables are stored within Lex and transmitted to the HubSpot API via Lambda functions written in Node.js. Hubsy’s infrastructure was all managed via terraform and he is deployed on Facebook, Slack and AWS Alexa.
